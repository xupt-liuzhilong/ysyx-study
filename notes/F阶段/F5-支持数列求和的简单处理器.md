# 学习记录：F5 支持数列求和的简单处理器

## 一、学习概况

今天开始并基本完成一生一芯 F5 阶段：支持数列求和的简单处理器。

F5 阶段的目标是把 F3 中学习的数字电路模块和 F4 中学习的状态机模型结合起来，设计一个能够执行简单指令序列的 sCPU。

本阶段最终实现了一个可以执行数列求和程序的简单处理器，支持三条指令：

```text
li
add
bner0
```

最终运行结果正确：

```text
PC = 7
r0 = 10
r1 = 10
r2 = 55
r3 = 1
```

其中：

```text
r2 = 55
```

表示程序成功计算出了：

```text
1 + 2 + 3 + ... + 10 = 55
```

------

## 二、sCPU 总体结构

F5 中实现的简单处理器主要由以下部分组成：

```text
PC 寄存器
ROM 指令存储器
指令译码逻辑
GPR 通用寄存器堆
adder8 加法器
比较器
PC 选择逻辑
控制信号生成逻辑
```

整体执行流程为：

```text
PC 给出当前指令地址
↓
ROM 根据 PC 输出当前指令 instr
↓
译码 instr，得到 opcode、rd、rs1、rs2、imm、addr
↓
根据指令类型执行 li / add / bner0
↓
写回寄存器或更新 PC
```

这对应 F4 中理解的状态机模型：

```text
当前状态
↓
组合逻辑计算下一状态
↓
时钟上升沿更新状态
```

## 三、sISA 指令格式

本阶段使用的指令宽度为 8 位。

### 1. add 指令

格式：

```text
00 rd rs1 rs2
```

功能：

```text
R[rd] = R[rs1] + R[rs2]
PC = PC + 1
```

例如：

```text
17 = 00010111
```

拆分为：

```text
opcode = 00
rd     = 01
rs1    = 01
rs2    = 11
```

表示：

```text
add r1, r1, r3
```

------

### 2. li 指令

格式：

```text
10 rd imm
```

功能：

```text
R[rd] = imm
PC = PC + 1
```

其中 `imm` 为 4 位立即数，写入 8 位寄存器时高 4 位补 0：

```text
wdata = 0000_imm
```

例如：

```text
8A = 10001010
```

拆分为：

```text
opcode = 10
rd     = 00
imm    = 1010
```

表示：

```text
li r0, 10
```

------

### 3. bner0 指令

格式：

```text
11 addr rs2
```

功能：

```text
if (R[0] != R[rs2])
    PC = addr;
else
    PC = PC + 1;
```

例如：

```text
D1 = 11010001
```

拆分为：

```text
opcode = 11
addr   = 0100
rs2    = 01
```

表示：

```text
bner0 r1, 4
```

含义是：

```text
如果 r0 != r1，则跳转到 PC=4
否则 PC = PC + 1
```

## 五、第一步：取指模块 Fetch

首先实现了取指模块：

```text
PC → ROM → instr
```

PC 使用 4 位寄存器 `reg4` 保存。

PC 默认更新规则为：

```text
PC_next = PC + 1
```

电路结构为：

```text
PC_reg.Q → adder4.A
0001     → adder4.B
adder4.S → PC_reg.D
```

ROM 设置为：

```text
Address Bit Width = 4
Data Bit Width = 8
```

测试结果：

| PC   | ROM 输出 |
| ---- | -------- |
| 0    | 8A       |
| 1    | 90       |
| 2    | A0       |
| 3    | B1       |
| 4    | 17       |
| 5    | 29       |
| 6    | D1       |
| 7    | DF       |
| 8~15 | 00       |

由于 PC 是 4 位，因此 PC 从 0 到 15 后会回到 0。

取指模块测试通过，说明：

```text
PC 自动加 1 正常
ROM 根据 PC 输出指令正常
```

## 六、第二步：实现 gpr4x8 寄存器堆

为了支持指令执行，实现了 4 个 8 位通用寄存器组成的寄存器堆：

```text
r0
r1
r2
r3
```

封装模块名：

```text
gpr4x8
```

### 1. gpr4x8 接口

输入：

```text
CLK
wen
waddr[1:0]
wdata[7:0]
raddr1[1:0]
raddr2[1:0]
```

输出：

```text
rdata1[7:0]
rdata2[7:0]
r0[7:0]
r1[7:0]
r2[7:0]
r3[7:0]
```

其中：

- `wen`：写使能
- `waddr`：写寄存器地址
- `wdata`：写入数据
- `raddr1`：读端口 1 地址
- `raddr2`：读端口 2 地址
- `rdata1`：读端口 1 数据
- `rdata2`：读端口 2 数据

------

### 2. 写入逻辑

写入逻辑为：

```text
waddr → 2-4 Decoder
```

得到：

```text
waddr = 00 → 选择 r0
waddr = 01 → 选择 r1
waddr = 10 → 选择 r2
waddr = 11 → 选择 r3
```

写使能：

```text
r0.EN = wen & rd_is_r0
r1.EN = wen & rd_is_r1
r2.EN = wen & rd_is_r2
r3.EN = wen & rd_is_r3
```

如果 Decoder 自带 Enable，则可直接把 `wen` 接入 Decoder 的使能端。

------

### 3. 读取逻辑

为了支持 `add rd, rs1, rs2`，寄存器堆需要同时读出两个寄存器。

读端口 1：

```text
rdata1 = R[raddr1]
```

读端口 2：

```text
rdata2 = R[raddr2]
```

实现方法：

```text
r0, r1, r2, r3 → 4选1 8位 MUX → rdata1
r0, r1, r2, r3 → 4选1 8位 MUX → rdata2
```

测试结果表明，`gpr4x8` 的写入和读取功能均正常。

## 八、第四步：实现 add 指令

`add` 指令格式为：

```text
00 rd rs1 rs2
```

实现逻辑：

```text
instr[5:4] → waddr
instr[3:2] → raddr1
instr[1:0] → raddr2
rdata1 → adder8.A
rdata2 → adder8.B
adder8.S → add_result
```

写回数据选择：

```text
li  指令：wdata = 0000_imm
add 指令：wdata = add_result
```

因此加入了一个 8 位 2 选 1 MUX：

```text
is_add = 0 → wdata = 0000_imm
is_add = 1 → wdata = add_result
```

指令类型判断：

```text
is_add = ~instr7 & ~instr6
is_li  =  instr7 & ~instr6
wen    = is_add | is_li
```

测试地址 4 和地址 5 的 add 指令：

```text
4: add r1, r1, r3
5: add r2, r2, r1
```

执行结果：

| 时钟后 | PC   | instr | r0   | r1   | r2   | r3   |
| ------ | ---- | ----- | ---- | ---- | ---- | ---- |
| 4      | 4    | 17    | 10   | 0    | 0    | 1    |
| 5      | 5    | 29    | 10   | 1    | 0    | 1    |
| 6      | 6    | D1    | 10   | 1    | 1    | 1    |

说明 `add` 指令执行正确。

## 九、第五步：实现 bner0 分支指令

`bner0` 指令格式为：

```text
11 addr rs2
```

功能：

```text
if (R[0] != R[rs2])
    PC = addr;
else
    PC = PC + 1;
```

实现逻辑：

```text
is_bne = instr7 & instr6
branch_addr = instr[5:2]
rs2 = instr[1:0]
rdata2 = R[rs2]
```

比较逻辑：

```text
neq = (r0 != rdata2)
```

跳转条件：

```text
branch_taken = is_bne & neq
```

PC 下一值选择：

```text
branch_taken = 0 → PC_next = PC + 1
branch_taken = 1 → PC_next = branch_addr
```

也就是：

```text
PC_next = branch_taken ? branch_addr : PC + 1
```

如果 `CLR = 1`，则优先清零：

```text
PC_D = CLR ? 0000 : PC_next
```

## 十、bner0 调试问题

在调试 `bner0 r1, 4` 时，最开始 PC 没有跳回 4，而是继续加到 7。

问题原因是：

自己现场搭建的比较器输出逻辑是：

```text
相同 → 0
不同 → 1
```

也就是说该比较器输出本身就是：

```text
neq
```

但是最开始又对这个比较结果取了一次反，导致：

```text
r0 != rdata2 时，neq 被反成 0
```

于是：

```text
branch_taken = is_bne & neq
             = 1 & 0
             = 0
```

因此 PC 继续执行：

```text
PC = PC + 1
```

修正方法：

```text
branch_taken = is_bne & compare_out
```

不要再对 `compare_out` 取反。

修正后，执行到：

```text
PC = 6
instr = D1
```

此时：

```text
r0 = 10
r1 = 1
r0 != r1
```

所以：

```text
branch_taken = 1
PC_next = 4
```

分支跳转成功。

## 十一、完整程序运行结果

完整程序运行过程：

```text
0: li r0, 10
1: li r1, 0
2: li r2, 0
3: li r3, 1
4: add r1, r1, r3
5: add r2, r2, r1
6: bner0 r1, 4
7: bner0 r3, 7
```

循环部分为：

```text
4: add r1, r1, r3
5: add r2, r2, r1
6: bner0 r1, 4
```

含义：

```text
r1 = r1 + 1
r2 = r2 + r1
如果 r0 != r1，则跳回 PC=4
```

最终：

```text
r1 = 10
r2 = 55
```

此时执行：

```text
6: bner0 r1, 4
```

因为：

```text
r0 = 10
r1 = 10
r0 == r1
```

所以不再跳回 4，而是执行：

```text
PC = PC + 1
```

进入：

```text
PC = 7
```

地址 7 的指令是：

```text
7: bner0 r3, 7
```

因为：

```text
r0 = 10
r3 = 1
r0 != r3
```

所以一直跳回：

```text
PC = 7
```

这相当于用自跳转死循环模拟程序结束。

最终状态：

```text
PC = 7
r0 = 10
r1 = 10
r2 = 55
r3 = 1
```

结果正确。

## 十二、F5 阶段核心理解

F5 的本质是：

```text
用数字电路实现 ISA 状态机
```

在 F4 中，ISA 状态机的状态为：

```text
PC + GPR + 内存
```

在 F5 中，这些状态由硬件模块保存：

```text
PC 由 reg4 保存
GPR 由 gpr4x8 保存
程序由 ROM 保存
```

每条指令的语义由组合逻辑实现：

```text
li    → 立即数扩展并写寄存器
add   → 读寄存器，ALU 加法，写回寄存器
bner0 → 比较寄存器，决定 PC 是否跳转
```

所以程序运行过程就是：

```text
ROM 输出指令
↓
译码产生控制信号
↓
数据通路计算结果
↓
时钟上升沿更新 PC 和 GPR
```
