## 二、处理器执行程序的基本过程

计算机执行程序时，并不是“直接理解 C 语言”，而是执行一条条机器指令。

处理器执行程序的核心过程可以概括为：

```text
取指令
执行指令
更新 PC
```

其中：

- `PC`：程序计数器，表示当前或下一条要执行的指令位置
- `GPR`：通用寄存器，用来保存临时数据
- `指令`：告诉处理器要进行什么操作
- `ISA`：指令集架构，规定每条指令的含义和执行效果

在 F4 文档中的简单 ISA 模型里，有 4 个通用寄存器：

```text
r0, r1, r2, r3
```

处理器的状态可以用下面这种形式表示：

```text
(PC, r0, r1, r2, r3)
```

每执行一条指令，这个状态就会发生一次转移。

## 三、官方数列求和程序

官方给出的程序用于计算：

```text
1 + 2 + 3 + ... + 10
```

指令序列如下：

```text
0: li r0, 10
1: li r1, 0
2: li r2, 0
3: li r3, 1
4: add r1, r1, r3
5: add r2, r2, r1
6: bner0 r1, 4
7: bner0 r3, 7
```

各寄存器含义：

| 寄存器 | 作用         |
| ------ | ------------ |
| r0     | 保存上限 10  |
| r1     | 当前加到的数 |
| r2     | 累加和 sum   |
| r3     | 常数 1       |

循环部分是：

```text
4: add r1, r1, r3
5: add r2, r2, r1
6: bner0 r1, 4
```

含义是：

```text
r1 = r1 + 1
r2 = r2 + r1
如果 r1 != r0，则跳回 PC=4 继续循环
```

## 四、官方数列求和状态推导

状态用：

```text
(PC, r0, r1, r2, r3)

| 2    | 10   | 0    | 0    | 0    |

```

## 五、10 以内奇数求和程序

接着，我尝试用相同的 sISA 指令计算：

```text
1 + 3 + 5 + 7 + 9
```

目标结果是：

```text
25
```

设计思路：

| 寄存器 | 作用       |
| ------ | ---------- |
| r0     | 终止值 11  |
| r1     | 当前奇数   |
| r2     | 累加和 sum |
| r3     | 常数 2     |

这里 `r0` 设置为 11，而不是 9。

原因是程序每轮先加当前奇数，再让当前奇数加 2。加完 9 后，`r1` 会变成 11，此时停止循环。

程序如下：

```text
0: li r0, 11
1: li r1, 1
2: li r2, 0
3: li r3, 2
4: add r2, r2, r1
5: add r1, r1, r3
6: bner0 r1, 4
7: bner0 r3, 7
```

循环部分：

```text
4: add r2, r2, r1
5: add r1, r1, r3
6: bner0 r1, 4
```

含义是：

```text
sum = sum + 当前奇数
当前奇数 = 当前奇数 + 2
如果当前奇数 != 11，则继续循环
```

## 七、状态机的基本概念

状态机一般包含：

```text
状态集合
激励事件集合
状态转移规则
初始状态
```

在 F4 中，可以把计算机系统理解为状态机。

对于 ISA 状态机：

| 状态机要素   | ISA 中的对应内容                  |
| ------------ | --------------------------------- |
| 状态         | PC、GPR、内存                     |
| 激励事件     | 执行一条指令                      |
| 状态转移规则 | 指令语义                          |
| 初始状态     | 程序开始前的 PC、寄存器、内存状态 |

也就是说，处理器执行程序，本质上就是：

```text
按照指令语义，不断更新 PC、寄存器和内存状态
```

每条指令规定了一种状态转移。

## 八、C 程序、ISA 和数字电路的关系

F4 进一步说明：C 程序、ISA、数字电路都可以看作状态机。

### 1. C 程序状态机

在 C 程序中：

| 状态机要素   | C 程序中的对应内容       |
| ------------ | ------------------------ |
| 状态         | 变量的值 + 当前执行位置  |
| 激励事件     | 执行一条语句             |
| 状态转移规则 | C 语言语句的语义         |
| 初始状态     | 程序开始时变量和执行位置 |

例如：

```c
int x = 1;
int y = 2;
int z = x + y;
```

可以理解为变量状态一步步变化。

------

### 2. ISA 状态机

ISA 状态机中：

| 状态机要素   | ISA 中的对应内容 |
| ------------ | ---------------- |
| 状态         | PC、GPR、内存    |
| 激励事件     | 执行一条指令     |
| 状态转移规则 | 指令语义         |

C 程序经过编译后，会变成 ISA 指令序列。

所以：

```text
编译器的作用：
把 C 程序状态机翻译成 ISA 状态机
```

------

### 3. 数字电路状态机

在数字电路中：

| 状态机要素   | 数字电路中的对应内容               |
| ------------ | ---------------------------------- |
| 状态         | 寄存器、触发器等时序逻辑保存的信息 |
| 激励事件     | 时钟到来                           |
| 状态转移规则 | 组合逻辑计算出的下一状态           |
| 初始状态     | 复位后的状态                       |

也就是说：

```text
时序逻辑保存当前状态
组合逻辑计算下一状态
时钟让状态更新
```

CPU 的设计，本质上就是：

```text
用数字电路状态机实现 ISA 状态机
```

## 九、F4 的核心理解

今天最重要的理解是：

```text
程序不是神秘地“自动运行”，而是状态不断转移的过程。
```

从 C 程序到 CPU 电路，可以这样理解：

```text
C 程序
↓ 编译器
ISA 指令序列
↓ CPU 硬件实现
数字电路状态机
```

对应关系：

```text
C 程序的变量状态
↓
ISA 中的 PC、寄存器、内存状态
↓
数字电路中寄存器和触发器保存的状态
```

所以，计算机执行程序可以统一理解为：

```text
状态 + 状态转移规则
```

这也是 F4 阶段的核心。
