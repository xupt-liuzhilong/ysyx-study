# 学习记录：F3 时序逻辑入门与寄存器

## 一、学习概况

今天正式进入一生一芯 F3 阶段的时序逻辑部分。

在进入 D 触发器之前，我按照文档顺序，从最基础的存储结构开始学习：

```text
交叉配对反相器
→ S-R 锁存器
→ D 锁存器
→ 主从 D 触发器
→ 1 位带写使能寄存器
→ 4 位带写使能寄存器
```

通过今天的学习，我理解了组合逻辑和时序逻辑的区别，并初步掌握了寄存器是如何保存数据的。

------

## 二、组合逻辑和时序逻辑的区别

之前完成的加法器、减法器、ALU、译码器、编码器、MUX、比较器等都属于组合逻辑。

组合逻辑的特点是：

```text
输出只由当前输入决定
```

例如加法器中，只要 A、B 输入改变，输出结果就会随之改变。

而时序逻辑的特点是：

```text
输出不仅和当前输入有关，还和过去保存的状态有关
```

也就是说，时序逻辑具有“记忆”能力。

CPU 中的寄存器、PC、状态机等都需要时序逻辑来实现。

------

## 三、交叉配对反相器

交叉配对反相器由两个 NOT 门交叉反馈组成：

```text
Q    = ~Qbar
Qbar = ~Q
```

它有两个稳定状态：

| Q    | Qbar |
| ---- | ---- |
| 0    | 1    |
| 1    | 0    |

这个结构说明：只要有反馈，电路就可能形成稳定状态，从而保存 1 bit 信息。

但是交叉配对反相器没有外部控制输入，不能方便地指定保存 0 还是 1，所以后面引出了 S-R 锁存器。

------

## 四、S-R 锁存器

S-R 锁存器有两个控制输入：

```text
S：Set，置 1
R：Reset，清 0
```

输出：

```text
Q
Qbar
```

我使用两个 NOR 门搭建了 NOR 版本 S-R 锁存器。

功能表：

| R    | S    | Q    | Qbar | 状态        |
| ---- | ---- | ---- | ---- | ----------- |
| 1    | 0    | 0    | 1    | Reset，清零 |
| 0    | 1    | 1    | 0    | Set，置一   |
| 0    | 0    | 保持 | 保持 | 记忆        |
| 1    | 1    | 0    | 0    | 非法状态    |

其中最重要的是保持状态：

```text
R = 0
S = 0
```

此时锁存器不会改变输出，而是保持原来的 Q。

通过这个实验，我理解了锁存器的核心功能：保存状态。

------

## 五、D 锁存器

S-R 锁存器的问题是存在非法状态：

```text
R = 1
S = 1
```

为了解决这个问题，可以在 S-R 锁存器前面加入控制逻辑，得到 D 锁存器。

D 锁存器输入：

```text
D
WE
```

输出：

```text
Q
Qbar
```

其中 WE 表示 Write Enable，写使能。

控制关系：

```text
S = WE & D
R = WE & ~D
```

这样可以保证 S 和 R 不会同时为 1。

D 锁存器功能：

| WE   | D    | Q      |
| ---- | ---- | ------ |
| 0    | x    | 保持   |
| 1    | 0    | 写入 0 |
| 1    | 1    | 写入 1 |

通过实验发现：

```text
WE = 1 时，Q 跟随 D
WE = 0 时，Q 保持原值
```

所以 D 锁存器是电平敏感的。

------

## 六、主从 D 触发器

D 锁存器的问题是：

```text
只要 WE=1，D 怎么变，Q 就怎么变
```

而 CPU 中更常用的是边沿触发，也就是只在时钟边沿更新。

我使用两个 D 锁存器搭建了主从 D 触发器：

```text
D → Master D_latch → Slave D_latch → Q
```

控制关系：

```text
Master.WE = ~CLK
Slave.WE  = CLK
```

工作过程：

```text
CLK = 0：
Master 打开，Slave 关闭，D 可以进入 Master，但最终 Q 不变

CLK 从 0 变 1：
Master 关闭，Slave 打开，Master 保存的数据传给 Slave，最终 Q 更新

CLK = 1：
Master 关闭，D 的变化不会再影响 Master，Q 保持

CLK 从 1 变 0：
Slave 关闭，最终 Q 不更新
```

因此该结构实现了上升沿触发：

```text
只有 CLK 从 0 变 1 时，Q 才更新为 D
```

这就是 D 触发器和 D 锁存器的区别：

| 电路     | 更新条件          | 特点     |
| -------- | ----------------- | -------- |
| D 锁存器 | WE=1 的整个时间段 | 电平敏感 |
| D 触发器 | CLK 上升沿        | 边沿敏感 |

------

## 七、1 位带写使能寄存器 reg1

D 触发器每个时钟上升沿都会写入 D。

但真正的寄存器还需要控制：

```text
这次时钟上升沿允不允许写入
```

所以我在 D 触发器前面加入了一个 2 选 1 MUX，搭建了 1 位带写使能寄存器。

核心公式：

```text
D_in = EN ? D : Q
```

也就是说：

| EN   | MUX 选择 | CLK 上升沿后 |
| ---- | -------- | ------------ |
| 0    | Q 反馈   | 保持原值     |
| 1    | 外部 D   | 写入新值     |

结构：

```text
Q反馈 ─── 0
          \
           MUX ───→ D_ff.D
          /
D ─────── 1

EN 控制 MUX
CLK 接 D_ff.CLK
```

通过测试：

```text
EN = 1，D = 1，CLK 上升沿后 Q = 1
EN = 0，D = 0，CLK 上升沿后 Q 仍保持 1
EN = 1，D = 0，CLK 上升沿后 Q = 0
```

说明 reg1 的写入和保持功能正确。

------

## 八、4 位带写使能寄存器 reg4

在完成 reg1 后，我使用 4 个 reg1 并联，搭建了 4 位带写使能寄存器 reg4。

结构：

```text
D3 → reg1 → Q3
D2 → reg1 → Q2
D1 → reg1 → Q1
D0 → reg1 → Q0
```

四个 reg1 共用：

```text
CLK
EN
```

功能：

```text
EN = 1：CLK 上升沿写入 D3D2D1D0
EN = 0：CLK 上升沿保持原来的 Q3Q2Q1Q0
```

测试样例：

| EN   | D3D2D1D0 | CLK 上升沿后 Q3Q2Q1Q0 | 说明     |
| ---- | -------- | --------------------- | -------- |
| 1    | 0000     | 0000                  | 初始化   |
| 1    | 1010     | 1010                  | 写入数据 |
| 0    | 0101     | 1010                  | 保持原值 |
| 1    | 0101     | 0101                  | 再次写入 |

通过该实验，我理解了：

```text
reg1 保存 1 bit
reg4 保存 4 bit
```

寄存器本质上就是多个触发器并联，并加入写使能控制。

## 九、遇到的问题与解决

在搭建 reg1 时，我把 Clock 元件换成普通 Pin 后，电路出现红线并且测试没有反应。

屏幕上出现提示：

```text
Simulator paused: no signals changed
```

原因主要有：

1. Logisim 的 Auto-Propagate 没有正常开启。
2. 仿真状态出现异常，需要 Reset Simulation。
3. 带反馈的寄存器初始状态可能是未知状态。
4. 普通 Pin 作为 CLK 时，需要手动点击 Pin 实现 0→1 上升沿。

解决方法：

```text
Simulate → Auto-Propagate
Simulate → Reset Simulation
```

然后先初始化寄存器：

```text
EN = 1
D  = 0
CLK：0 → 1
```

让 Q 变成确定的 0 后，再继续测试写入和保持功能。

通过这个问题，我理解了反馈电路需要初始化，也进一步熟悉了 Logisim 的仿真控制。
