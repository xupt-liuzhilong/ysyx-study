## 二、4 位二进制与补码范围

4 位二进制共有 4 个 bit：

```text
b3 b2 b1 b0
```

其中：

- `b0` 是最低位
- `b3` 是最高位

如果按无符号数解释，4 位二进制范围是：

```text
0000 ~ 1111
```

也就是：

```text
0 ~ 15
```

如果按 4 位补码解释，范围是：

```text
-8 ~ +7
```

对应关系如下：

| 二进制 | 补码值 |
| ------ | -----: |
| 0000   |      0 |
| 0001   |      1 |
| 0010   |      2 |
| 0011   |      3 |
| 0100   |      4 |
| 0101   |      5 |
| 0110   |      6 |
| 0111   |      7 |
| 1000   |     -8 |
| 1001   |     -7 |
| 1010   |     -6 |
| 1011   |     -5 |
| 1100   |     -4 |
| 1101   |     -3 |
| 1110   |     -2 |
| 1111   |     -1 |

通过这一部分，我理解到：

> 同一串二进制，按无符号数解释和按补码解释，含义可能不同。

例如：

```text
1110
```

按无符号数解释是 `14`，按 4 位补码解释是 `-2`。

## 三、原码、反码与补码

### 1. 原码

原码使用最高位表示符号位：

```text
0 表示正数
1 表示负数
```

例如 4 位原码：

```text
+3 = 0011
-3 = 1011
```

原码的问题是存在两个 0：

```text
+0 = 0000
-0 = 1000
```

---

### 2. 反码

反码中，正数不变，负数在原码基础上符号位不变，数值位取反。

例如：

```text
+3 = 0011
-3 原码 = 1011
-3 反码 = 1100
```

反码同样存在两个 0：

```text
+0 = 0000
-0 = 1111
```

---

### 3. 补码

补码中，正数不变，负数可以通过：

```text
取反 + 1
```

得到。

例如 4 位补码表示 `-3`：

```text
+3 = 0011
取反 = 1100
加 1 = 1101
```

所以：

```text
-3 = 1101
```

补码最重要的作用是：

> 可以把减法转换成加法。

例如：

```text
A - B = A + (~B + 1)
```

这使得 CPU 可以主要依靠加法器完成加法和减法运算。

## 四、4 位加减法器 add_sub4

在已经完成的 `adder4` 基础上，我通过加入 4 个 XOR 门和一个控制信号 `Sub`，实现了 4 位加减法器。

核心公式为：

```text
S = A + (B XOR Sub) + Sub
```

其中：

| Sub  | 功能 | 实际运算           |
| ---- | ---- | ------------------ |
| 0    | 加法 | A + B              |
| 1    | 减法 | A + ~B + 1 = A - B |

当 `Sub=0` 时：

```text
B XOR 0 = B
Cin = 0
```

所以电路执行：

```text
A + B
```

当 `Sub=1` 时：

```text
B XOR 1 = ~B
Cin = 1
```

所以电路执行：

```text
A + ~B + 1
```

也就是：

```text
A - B
```

---

## 五、add_sub4 不是传统全减器

今天还区分了加减法器和全减器的区别。

传统全减器 Full Subtractor 处理的是一位减法：

```text
A - B - Bin
```

它有：

```text
输入：A、B、Bin
输出：Diff、Bout
```

核心是借位逻辑。

而本次实现的 `add_sub4` 是：

```text
输入：A[3:0]、B[3:0]、Sub
输出：S[3:0]、Cout
```

它不是通过借位方式完成减法，而是通过补码思想：

```text
A - B = A + (~B + 1)
```

所以它更准确的名字是：

```text
4 位加减法器 / 4-bit Adder-Subtractor
```

## 六、Cout 与 Overflow

今天重点理解了 `Cout` 和 `Overflow` 的区别。

### 1. Cout

`Cout` 是加法器最高位产生的进位输出，主要用于观察无符号运算是否产生进位。

例如：

```text
1111 + 0001 = 1 0000
```

对于 4 位加法器来说：

```text
S = 0000
Cout = 1
```

这表示无符号运算结果超出了 4 位范围。

---

### 2. Overflow

`Overflow` 用来判断有符号补码运算是否超出了当前位宽能够表示的范围。

4 位补码范围是：

```text
-8 ~ +7
```

例如：

```text
7 + 1
```

数学结果是：

```text
8
```

但 4 位补码无法表示 `+8`，实际结果会变成：

```text
0111 + 0001 = 1000
```

而 `1000` 在 4 位补码中表示 `-8`，因此发生了有符号溢出：

```text
Overflow = 1
```

所以可以理解为：

> Overflow 不是电路算错了，而是当前位数不够，真正的数学结果装不下了。

---

## 七、Overflow 判断公式

对于本次加减法器，实际参与加法的第二个操作数为：

```text
B_eff = B XOR Sub
```

因此最高位为：

```text
B_eff3 = B3 XOR Sub
```

Overflow 的判断公式为：

```text
Overflow = ~(A3 XOR B_eff3) AND (A3 XOR S3)
```

含义是：

```text
A 和实际参与加法的 B_eff 符号相同，
但是结果 S 的符号和 A 不同，
则发生 Overflow。
```

也可以理解为：

```text
A3 == B_eff3，并且 S3 != A3
```

常见情况：

| 运算情况           | 是否溢出 |
| ------------------ | -------- |
| 正数 + 正数 = 负数 | 溢出     |
| 负数 + 负数 = 正数 | 溢出     |
| 正数 - 负数 = 负数 | 溢出     |
| 负数 - 正数 = 正数 | 溢出     |

## 九、简易 4 位 ALU 设计

在 `add_sub4` 基础上，我进一步设计了一个简易 4 位 ALU。

ALU 全称是：

```text
Arithmetic Logic Unit
```

即：

```text
算术逻辑单元
```

它负责 CPU 中的基本算术和逻辑运算。

本次设计的简易 ALU 支持 4 种运算：

| Op1  | Op0  | 功能   |
| ---- | ---- | ------ |
| 0    | 0    | A + B  |
| 0    | 1    | A - B  |
| 1    | 0    | A & B  |
| 1    | 1    | A \| B |

输入：

```text
A3 A2 A1 A0
B3 B2 B1 B0
Op1 Op0
```

输出：

```text
S3 S2 S1 S0
Cout
Overflow
Zero
Negative
```

## 十、ALU 的核心结构

通过本次实验，我理解了 ALU 的基本结构：

> 多条运算通路同时计算，最后通过 MUX 选择一路作为最终结果输出。

本次 ALU 中主要有三类运算通路：

```text
加减法通路：add_sub4
AND 通路：A & B
OR 通路：A | B
```

其中：

- `add_sub4` 负责 `A+B` 和 `A-B`
- AND 门负责按位与
- OR 门负责按位或
- 4 选 1 MUX 根据 `Op1 Op0` 选择最终输出结果

---

## 十一、4 选 1 MUX 的理解

4 选 1 MUX 有 4 路输入：

```text
D0 D1 D2 D3
```

有 2 位选择信号：

```text
S1 S0
```

选择规则为：

| S1   | S0   | 选择 |
| ---- | ---- | ---- |
| 0    | 0    | D0   |
| 0    | 1    | D1   |
| 1    | 0    | D2   |
| 1    | 1    | D3   |

在 ALU 中，`Op1 Op0` 就是 MUX 的选择信号。

以最低位 `S0` 为例，MUX 的输入为：

```text
D0 = add_sub4.S0
D1 = add_sub4.S0
D2 = A0 & B0
D3 = A0 | B0
```

之所以 `D0` 和 `D1` 都接 `add_sub4.S0`，是因为 `add_sub4` 内部已经根据 `Op0` 控制加法或减法：

```text
Op = 00 时，add_sub4 执行 A + B
Op = 01 时，add_sub4 执行 A - B
```

所以 MUX 只需要在加减法结果、AND 结果和 OR 结果之间选择即可。

---

## 十二、Splitter 与 MUX 选择端

Logisim 中的 MUX 设置为：

```text
Selection Bits = 2
Data Bits = 1
```

表示：

```text
选择信号宽度为 2 bit
每一路输入数据宽度为 1 bit
```

所以这个 MUX 是：

```text
4 选 1 的 1 位 MUX
```

虽然 MUX 看起来只有一个选择端口，但这个端口实际上是 2 位宽的总线。

因此需要使用 Splitter 将 `Op0` 和 `Op1` 合成为 2 位选择信号：

```text
Op0 → bit0
Op1 → bit1
```

也就是：

```text
Sel[0] = Op0
Sel[1] = Op1
```

这样 MUX 的选择关系才是：

| Op1  | Op0  | 选择 |
| ---- | ---- | ---- |
| 0    | 0    | D0   |
| 0    | 1    | D1   |
| 1    | 0    | D2   |
| 1    | 1    | D3   |

## 十三、Zero 和 Negative 标志位

### 1. Zero

`Zero` 表示最终 ALU 输出结果是否为 0。

公式为：

```text
Zero = ~(S3 | S2 | S1 | S0)
```

当结果为：

```text
0000
```

时：

```text
Zero = 1
```

否则：

```text
Zero = 0
```

---

### 2. Negative

`Negative` 表示最终输出结果是否为负数。

在补码表示中，最高位是符号位，因此：

```text
Negative = S3
```

如果：

```text
S3 = 1
```

则结果按补码解释为负数。

如果：

```text
S3 = 0
```

则结果按补码解释为非负数。
